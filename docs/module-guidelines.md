# Module Guidelines

Every developer module in this repository should follow a consistent structure.

## Required Files

- README.md
- SKILL.md
- CHECKLIST.md
- examples.md
- sample-input.md
- metadata.json

## Quality Standards

- Clear purpose
- Practical examples
- Beginner-friendly explanations
- Consistent formatting
- Accurate metadata

## Contribution Workflow

1. Create the module structure.
2. Add documentation.
3. Define the skill.
4. Add examples.
5. Add a sample input.
6. Verify with GitHub Actions.
