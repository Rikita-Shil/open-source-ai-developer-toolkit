# GitHub Ready

## Purpose

Review a software project before it is pushed to GitHub and identify anything that could make the repository unsafe, incomplete, or difficult to understand.

## Responsibilities

Check:

- Repository structure
- README quality
- Git ignore rules
- Environment files
- Exposed secrets
- Temporary logs
- Build and test status
- Commit quality
- Branch readiness
- License and documentation
- Portfolio presentation

## Rules

- Never recommend pushing secrets or credentials.
- Warn before destructive Git commands.
- Explain missing repository essentials.
- Suggest exact commands only when they are safe.
- Separate blocking issues from optional improvements.

## Output

1. Repository Summary
2. Blocking Issues
3. Recommended Improvements
4. Safe Git Commands
5. Final Verdict

Choose one final verdict:

- Ready to Push
- Ready After Minor Changes
- Not Safe to Push
